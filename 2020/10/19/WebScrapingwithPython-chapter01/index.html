《Python网络爬虫权威指南》

1.1 网络连接

#查找Python的request模块（在urllib库中），只导入urlopen函数
from urllib.request import urlopen
html = urlopen(“http://pythonscraping.com/pages/page1.html")
print(html.read())

urlopen用来打开并读取一个从网络获取的远程对象（可以轻松打开html文件，图像文件或者其他任何文件流）。

1.2 BeautifulSoup简介

通过定位HTML标签来格式化和组织复杂的网页信息，用简单的Python对象为我们展现XML结构信息。BeautifulSoup库常用的对象恰好是BeautifulSoup对象：

from urllib.request import urlopen
from bs4 import BeautifulSoup
html = urlopen(“http://pythonscraping.com/pages/page1.html")
bs = BeautifulSoup(html.read(), ‘html.parser’)
print(bs.h1)

当创建一个BeautifulSoup对象时，需要传入两个参数：

bs = BeautifulSoup(html.read(), ‘html.parser’)

第一个参数是该对象所基于的HTML文本，第二个参数指定了你希望BeautifulSoup用来创建该对象的解析器。html.parser是Python3自带的解析器，另一个常用的是lxml。lxml和html.parser相比的优点在于解析“杂乱”或者包含错误语法的HTML代码的性能更优一些，但是缺点是必须单独安装，且依赖于第三方的C语言库。另一个常用的HTML解析器是html5lib，也是具有容错性的解析器，但是同样需要外部依赖。

可靠的网络连接以及异常处理

html = urlopen(“http://pythonscraping.com/pages/page1.html")

这行代码主要会发生两种异常：

  • 网页在服务器上不存在（或者获取页面的时候出现错误）
  • 服务器不存在

第一种异常出现时，程序会返回HTTP错误，HTTP错误可能是“404 Page Not Found” “500 Internet Server Error”等。urlopen函数会抛出HTTPError异常，可以用下面的方式处理这些异常：

from urllib.request import urlopen
from urllib.error import HTTPError
try:
html = html = urlopen(“http://pythonscraping.com/pages/page1.html")
except HTTPError as e:
print(e)
#返回空值，中断程序
else:
#程序继续。注意：如果已经在上面异常捕捉那一段代码中返回或者中断（break）
#那么就不需要再使用else语句了，这段代码也不会执行

如果程序返回HTTP错误代码，程序就会显示错误内容，不再执行else后面的语句

第二种异常出现时（链接打不开或者URL链接写错了），urlopen会抛出一个URLError异常。这意味着获取不到服务器，可以检查下面代码：

from urllib.request import urlopen
from urllib.error import HTTPError
from urllib.error import URLError
try:
html = urlopen(“http://pythonscraping.com/pages/page1.html")
except HTTPError as e:
print(e)
except URLError as e:
print(“The server could not be found!”)
else:
print(“It Worked!”)

当调用BeautifulSoup对象里面的一个标签时，增加一个检查条件以保证标签确实存在是很聪明的做法，但想调用的标签不存在时，BeautifulSoup就会返回None对象。

print(bs.nonExitentTag)

会返回一个None对象，如果不检查而直接调用这个None对象里面的子标签，会有麻烦，如下：

print(bs.nonExistentTag.someTag)

此时会返回一个异常：

AttributeError: ‘NoneType’ object has no attribute ‘someTag’

为避免这两种异常情形，最简单的方法是对这两种情形进行检查：

try:
badContent = bs.nonExistingTag.anotherTag
except AttributeError as e:
print(‘Tag was not found’)
else:
if badContent == None:
print(‘Tag was not found’)
else:
print(badContent)

综合以上错误类型以及检查代码，可以将之前的爬虫代码重新书写：

from urllib.request import urlopen
from urllib.error import HTTPError
from bs4 import BeautifulSoup
def getTitle(url):
try:
html = urlopen(url)
except HTTPError as e:
return None
try:
bs = BeautifulSoup(html.read(), ‘html.parser’)
title = bs.body.h1
except AttributeError as e:
return None
return title
title = getTitle(‘http://www.pythonscraping.com/pages/page1.html')
if title == None:
print(‘Title could not be found’)
else:
print(title)

Title could not be found

上面我们创建了一个getTitle函数，可以返回网页的标题，在获取网页的过程中遇见问题则会返回一个None对象，在getTitle函数里面，检查了HTTRError，和由于URL输入错误引起的URLError，再将这两行封装在一个try语句中。这两行中的任何一行有问题，都可能抛出AttributeError（如果服务器不存在，html就是一个None对象，html.read()就会抛出AttributeError），可以在try中放入任意多行代码，或者调用一个在任意位置都可以抛出AttributeError的函数。
